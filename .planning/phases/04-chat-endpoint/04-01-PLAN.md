**TDD Feature: SSE streaming chat handler**

This is the core implementation. Follow RED-GREEN-REFACTOR.

**Request/response types** (in chat.go):

- `ChatRequest` with `Messages []ChatMessage`
- `ChatMessage` with `Role string` and `Content string`
- `TokenPayload` with `Text string` and `Seq int`
- `DonePayload` with `StopReason string` and `Usage UsageStats`
- `UsageStats` with `InputTokens int64` and `OutputTokens int64`
- `ErrorPayload` with `Code string` and `Message string`
- `maxChatMessages = 50` const
- `maxRequestBodySize = 1 << 20` const (1 MB)

**writeSSEEvent helper** (in chat.go):

- `writeSSEEvent(w http.ResponseWriter, flusher http.Flusher, eventType string, data any) error`
- Format: `event: {type}\ndata: {json}\n\n`
- Calls `flusher.Flush()` after writing

**buildChatSystemPrompt method** (on Handler):

- Build `ai.ContextInput` from handler state:
  - `BOSHConfigured: h.boshClient != nil`
  - `VSphereConfigured: h.cfg.VSphereConfigured()`
  - Dashboard from cache (`h.cache.Get("dashboard:all")`)
  - Infrastructure from `h.infrastructureState` (read-locked via `h.infraMutex.RLock()`)
  - `LogCacheAvailable`: derive from dashboard -- true if any app has `ActualMB > 0`
- Call `ai.BuildContext(input)` then `ai.BuildSystemPrompt(context)`

**Chat handler method** (on Handler):
Phase 1 -- Pre-stream validation (returns JSON errors via `h.writeError`):

1. Check `h.chatProvider != nil` -> 503 "AI advisor not configured"
2. Decode request body with `http.MaxBytesReader` (maxRequestBodySize) -> 400 on error
3. Validate `len(req.Messages) == 0` -> 400 "Messages required"
4. Validate `len(req.Messages) > maxChatMessages` -> 400 "Maximum 50 messages per request"
5. Validate each message has non-empty content and valid role ("user" or "assistant") -> 400

Phase 2 -- Context snapshot: 6. Call `h.buildChatSystemPrompt()` to snapshot current infrastructure state

Phase 3 -- SSE streaming: 7. Assert `http.Flusher` -> 500 if not supported 8. Set SSE headers: `Content-Type: text/event-stream`, `Cache-Control: no-cache`, `Connection: keep-alive`, `X-Accel-Buffering: no` 9. Disable write deadline via `http.NewResponseController(w).SetWriteDeadline(time.Time{})` 10. Create cancellable context wrapping `r.Context()` 11. Convert `ChatMessage` to `ai.Message` slice 12. Call `h.chatProvider.Chat(ctx, messages, ai.WithSystem(systemPrompt))` 13. Streaming loop with select on token channel and `ctx.Done()`: - On `TokenEvent` with text: write `token` SSE event with `{text, seq}` (1-based seq) - On `TokenEvent` with `Done: true`: write `done` SSE event with `{stop_reason, usage}`, return - On `TokenEvent` with `Err != nil`: write `error` SSE event with `{code: "provider_error", message}`, log at `slog.Warn`, return - On `ctx.Done()`: return (client disconnect, no one to send to) - On channel closed without done: return

Note: Idle timeout and max duration timers are Plan 02 scope. This plan implements the basic streaming loop without timeout logic.
