**TDD Feature: Idle timeout, max duration, and client disconnect**

Follow RED-GREEN-REFACTOR. Modify the streaming loop in the Chat handler (from Plan 01) to add timeout protection.

**Streaming loop changes** (in chat.go Chat method):

Replace the basic streaming loop with a timeout-aware version:

1. Create a `maxDurationExceeded` channel (or `atomic.Bool`) to distinguish max-duration cancel from client disconnect
2. Start max duration timer: `time.AfterFunc(time.Duration(h.cfg.AIMaxDurationSecs) * time.Second, func() { close(maxDurationExceeded); cancel() })`
3. Defer `maxDuration.Stop()`
4. Create idle timer: `time.NewTimer(time.Duration(h.cfg.AIIdleTimeoutSecs) * time.Second)`
5. Defer `idleTimer.Stop()`

In the select loop, add two new cases:

- `case <-idleTimer.C:` -- write SSE error event `{code: "timeout", message: "No response from AI provider within timeout window"}`, return
- `case <-ctx.Done():` -- check if `maxDurationExceeded` channel is closed (non-blocking select). If max duration: write SSE error event `{code: "timeout", message: "Response exceeded maximum duration"}`, return. If client disconnect: just return (no one to send to)

On each token received, reset the idle timer using the safe drain pattern:

```go
if !idleTimer.Stop() {
    select {
    case <-idleTimer.C:
    default:
    }
}
idleTimer.Reset(idleTimeout)
```

**Test cases** (add to chat_test.go):
