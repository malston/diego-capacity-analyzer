  <behavior>
    Interface contract:
    - ChatProvider has a single method: Chat(ctx context.Context, messages []Message, opts ...Option) <-chan TokenEvent
    - Message has exactly two fields: Role string, Content string
    - TokenEvent has fields: Text string, Done bool, StopReason string, Usage *Usage, Err error
    - Usage has fields: InputTokens int64, OutputTokens int64

    Option functions:
    - WithMaxTokens(4096) -> ChatConfig.MaxTokens == 4096
    - WithTemperature(0.3) -> ChatConfig.Temperature points to 0.3
    - WithSystem("prompt") -> ChatConfig.System == "prompt"
    - WithModel("claude-3") -> ChatConfig.Model == "claude-3"
    - No options applied -> ChatConfig has zero values (defaults resolved by provider implementation)
    - Multiple options compose: WithMaxTokens(1024) + WithTemperature(0.5) -> both set

    ChatConfig resolution:
    - NewChatConfig(opts ...Option) applies all options in order
    - Last option wins for same field

  </behavior>
  <implementation>
    1. Create backend/services/ai/ directory and package `ai`
    2. provider.go: Define ChatProvider interface, Message, TokenEvent, Usage, ChatConfig types. Add NewChatConfig function that applies options. Follow ABOUTME convention. System prompt is per-request via WithSystem (per CONTEXT.md discretion recommendation from RESEARCH.md).
    3. options.go: Define Option type (func(*ChatConfig)) and constructor functions. Keep Model field as string (not SDK-specific type).
    4. Tests use standard Go testing with table-driven patterns per project conventions. No third-party assertion libraries needed for these simple equality checks.
  </implementation>
