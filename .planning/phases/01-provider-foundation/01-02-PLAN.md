  <behavior>
    Constructor:
    - NewAnthropicProvider(apiKey string, defaults ChatConfig) *AnthropicProvider
    - Creates anthropic.Client using option.WithAPIKey(apiKey)
    - Stores default ChatConfig (model, maxTokens, temperature) for use when options omit them
    - Default model: anthropic.ModelClaude3_7SonnetLatest (per research recommendation)
    - Default temperature: 0.3 (deterministic infrastructure advice)
    - Default max tokens: 4096

    Chat method:
    - Resolves options over defaults (NewChatConfig merges; provider fills remaining defaults)
    - Converts domain []Message to []anthropic.MessageParam via toSDKMessages helper
    - System prompt (from WithSystem option) converted to []anthropic.TextBlockParam
    - Calls client.Messages.NewStreaming(ctx, params)
    - Spawns goroutine that: reads stream.Next(), extracts TextDelta tokens, sends TokenEvent to buffered(1) channel
    - Uses accumulated.Accumulate(event) to track usage
    - On stream completion: logs usage via slog.Info, sends final TokenEvent{Done: true, StopReason, Usage}
    - On error: sends TokenEvent{Err: err, Done: true}
    - All channel sends use select with ctx.Done() to prevent goroutine leaks
    - defer stream.Close() after stream creation

    Message mapping (toSDKMessages):
    - "user" role -> anthropic.NewUserMessage(anthropic.NewTextBlock(content))
    - "assistant" role -> anthropic.NewAssistantMessage(anthropic.NewTextBlock(content))
    - "system" role messages are skipped (system prompt passed separately via WithSystem)
    - Unknown roles are skipped with slog.Warn

    Config resolution (resolveConfig):
    - Apply user options via NewChatConfig
    - Fill zero-value fields from provider defaults
    - MaxTokens 0 -> provider default (4096)
    - Temperature nil -> provider default (0.3)
    - Model "" -> provider default model

    Send helper:
    - send(ctx, ch, event) uses select { case ch <- event: case <-ctx.Done(): }

  </behavior>
