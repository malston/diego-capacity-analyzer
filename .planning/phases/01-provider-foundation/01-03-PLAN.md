    Add AI provider initialization block in main.go after the session service initialization and before the JWKS client setup (so it runs early in startup). Follow the existing nil-service pattern used for BOSH and vSphere.

    Logic:
    ```
    if cfg.AIProvider == "" {
        slog.Info("AI provider not configured, advisor feature disabled")
    } else if cfg.AIProvider == "anthropic" {
        if cfg.AIAPIKey == "" {
            slog.Error("AI_API_KEY required when AI_PROVIDER is set", "provider", cfg.AIProvider)
            os.Exit(1)
        }
        chatProvider := ai.NewAnthropicProvider(cfg.AIAPIKey, ai.ChatConfig{})
        h.SetChatProvider(chatProvider)  // Note: h must be created before this -- may need to reorder
        slog.Info("AI provider initialized", "provider", cfg.AIProvider)
    } else {
        slog.Error("Unknown AI_PROVIDER value", "provider", cfg.AIProvider)
        os.Exit(1)
    }
    ```

    **Important ordering consideration:** The Handler `h` is created after services are initialized. The AI provider initialization should go after `h := handlers.NewHandler(cfg, c)` and `h.SetSessionService(sessionService)`, similar to how SetSessionService is called after construction. Place the AI provider block right after the SetSessionService call.
