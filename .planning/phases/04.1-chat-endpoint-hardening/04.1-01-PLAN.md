**Auth guard (chat.go):**

4. At the very TOP of Chat(), before the nil provider check, add:

```go
claims := middleware.GetUserClaims(r)
if claims == nil {
    h.writeError(w, "authentication required for AI advisor", http.StatusUnauthorized)
    return
}
```

5. Add `middleware` import to chat.go.
6. Change `buildChatSystemPrompt()` signature to `buildChatSystemPrompt(username string)`.
7. In Chat(), pass `claims.Username` to buildChatSystemPrompt: `systemPrompt := h.buildChatSystemPrompt(claims.Username)`.

**Scenario wiring (chat.go + scenario.go):**

8. In `buildChatSystemPrompt(username string)`, after the infraMutex read, add:

```go
h.userScenariosMutex.RLock()
input.Scenario = h.userScenarios[username]
h.userScenariosMutex.RUnlock()
```

IMPORTANT: Acquire/release infraMutex BEFORE acquiring userScenariosMutex. Never hold both locks simultaneously (deadlock prevention per RESEARCH.md Pitfall 2).

9. In CompareScenario (scenario.go), after computing the comparison and before writeJSON, add:

```go
claims := middleware.GetUserClaims(r)
if claims != nil {
    h.userScenariosMutex.Lock()
    h.userScenarios[claims.Username] = &comparison
    h.userScenariosMutex.Unlock()
}
```

Add `middleware` import to scenario.go.
