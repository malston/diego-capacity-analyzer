GREEN phase: Implement `BuildContext` to pass all tests.

Top-level function uses `strings.Builder`, calls section writers in order per locked decision:

1. `writeDataSourceSummary(b, input)` -- Data source status block at top
2. `writeInfrastructure(b, input.Infra, input.VSphereConfigured)` -- vSphere cluster data or marker
3. `writeDiegoCells(b, input.Dashboard)` -- Per-segment cell aggregates
4. `writeApps(b, input.Dashboard)` -- Top-N apps by memory
5. `writeScenario(b, input.Scenario)` -- Before/after delta summary

Section writer conventions:

- Signature: `func writeXxx(b *strings.Builder, ...)` (unexported)
- Each writer emits its own `## Heading` and handles nil/missing data by writing the appropriate marker
- Two standard markers: "NOT CONFIGURED" (admin choice, source bool is false) vs "UNAVAILABLE" (configured but data missing/errored)
- Sections for unconfigured sources MUST still appear with the marker (never silently omitted)

**Data Source Summary** (`writeDataSourceSummary`):

- CF API: "available" if Dashboard non-nil and has apps/cells; "UNAVAILABLE" otherwise
- BOSH: "NOT CONFIGURED" if !BOSHConfigured; "UNAVAILABLE" if configured but Dashboard.Metadata.BOSHAvailable is false; "available" otherwise
- vSphere: "NOT CONFIGURED" if !VSphereConfigured; "UNAVAILABLE" if configured but Infra is nil; "available" otherwise
- Log Cache: "available" if LogCacheAvailable; "not available" otherwise (note: Log Cache is not independently configured, it rides on BOSH)

**Infrastructure** (`writeInfrastructure`):

- If !VSphereConfigured: write "NOT CONFIGURED" marker, return
- If Infra is nil but VSphereConfigured: write "UNAVAILABLE" marker, return
- One-line intro: "Physical hosts and clusters backing Diego cells."
- For each cluster: name, host count, total memory GB, HA usable memory GB, HA status, host memory utilization %, vCPU:pCPU ratio
- Use inline threshold flags: utilization >80% gets " [HIGH]", >90% gets " [CRITICAL]"; vCPU ratio >8:1 gets " [HIGH]"
- Overall totals: total hosts, total memory, overall HA status

**Diego Cells** (`writeDiegoCells`):

- If Dashboard is nil: write "UNAVAILABLE" marker, return
- One-line intro: "Diego cell capacity grouped by isolation segment."
- Aggregate cells by IsolationSegment (empty string = "shared")
- Per segment: name, cell count, total memory MB, allocated MB, utilization % (allocated/total), used MB (if available from Log Cache)
- Flag utilization >80% with " [HIGH]", >90% with " [CRITICAL]"
- Overall totals: total cells, total memory, overall utilization

**Apps** (`writeApps`):

- If Dashboard is nil or no apps: write "UNAVAILABLE" marker, return
- One-line intro: "Top applications by memory allocation."
- Sort apps by RequestedMB descending, take top 10
- Per app: name, instances, requested MB, actual MB (if available), isolation segment
- Note total count: "Showing 10 of 47 applications"
- If any apps have ActualMB == 0 while others don't: "Note: Memory usage unavailable for N of M apps"

**Scenario Comparison** (`writeScenario`):

- If Scenario is nil: write "No scenario comparison has been run.", return
- One-line intro: "Current vs proposed capacity changes."
- Compact markdown table: Metric | Current | Proposed | Delta
- Rows: Cells, Cell Size, App Capacity (GB), Utilization %, Disk Capacity (GB) (if non-zero)
- Include warnings if any (as bullet list below table)

ABOUTME comments on context.go following codebase convention.
