Error type mapping (from RESEARCH.md):
| Source | Signal | Type |
|--------|--------|------|
| Pre-stream HTTP 429 | response.status === 429 | "rate_limit" |
| Pre-stream fetch TypeError | err instanceof TypeError | "network" |
| Mid-stream SSE code "timeout" | event.data.code === "timeout" | "timeout" |
| Mid-stream SSE code "provider_error" | event.data.code === "provider_error" | "server" |
| Any other error | Fallback | "server" |

  <action>
    **chatApi.js changes:**

    1. Create a `ChatError` class that extends Error with a `type` field. Export it.
       ```javascript
       export class ChatError extends Error {
         constructor(message, type = "server") {
           super(message);
           this.name = "ChatError";
           this.type = type;
         }
       }
       ```

    2. In the `streamChat` generator, replace the pre-stream error throw:
       - Check `response.status === 429` → throw `new ChatError(message, "rate_limit")`
       - All other non-OK statuses → throw `new ChatError(message, "server")`

    3. Wrap the `fetch()` call in a try/catch to detect `TypeError` (network errors):
       - If `err instanceof TypeError` → throw `new ChatError("Connection lost", "network")`
       - Re-throw other errors

    4. In the SSE event loop, when yielding `error` events, attach the SSE error code to the event data so the hook can classify mid-stream errors. The event already has `event.data.code` from the backend. No change needed to the yield -- the hook will read `event.data.code`.

    **useChatStream.js changes:**

    1. Import `ChatError` from chatApi.js.

    2. Change error state from `string | null` to `{ message: string, type: string } | null`.

    3. In the catch block of sendMessage:
       - If `err instanceof ChatError`: `setError({ message: err.message, type: err.type })`
       - If `err.name === "AbortError"`: skip (existing behavior)
       - Else: `setError({ message: err.message, type: "server" })`

    4. For mid-stream SSE errors (the `event.type === "error"` branch): instead of `throw new Error(event.data.message)`, create a ChatError:
       - Map `event.data.code`: "timeout" → "timeout", "provider_error" → "server", anything else → "server"
       - `throw new ChatError(event.data.message, mappedType)`

    5. Add `clearConversation` function (useCallback):
       - Abort active stream: `abortRef.current?.abort(); abortRef.current = null;`
       - Reset state: `setMessages([]); setIsStreaming(false); setError(null);`

    6. Add `retryLastMessage` function (useCallback):
       - If messages is empty, return early (no-op)
       - Find the last user message content from messages array (search backwards for role === "user")
       - If no user message found, return early
       - Remove the last assistant message from the array: `setMessages(prev => { ... filter out last assistant message ... })`
       - Clear error: `setError(null)`
       - Call `sendMessage(lastUserContent)`
       - NOTE: Use a ref for messages (messagesRef.current is already maintained) to avoid stale closure. Derive last user message from messagesRef.current, not from the messages state variable.

    7. Update the hook return value:
       ```javascript
       return { messages, isStreaming, error, sendMessage, clearConversation, retryLastMessage };
       ```
