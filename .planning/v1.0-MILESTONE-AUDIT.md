### Missing Connection: MISS-01

**Scenario comparison never reaches AI context**

- `ContextInput.Scenario` field exists (Phase 2) but is never populated by `buildChatSystemPrompt` (Phase 4)
- `CompareScenario` in `handlers/scenario.go` computes and returns results without caching
- The `Handler` struct has no field to store the last scenario result
- Result: AI always sees "No scenario comparison has been run" regardless of actual state
- **Affected requirement:** CTX-03 (per-phase satisfied; E2E flow broken)

## Tech Debt (3 items, 0 blockers)

### Phase 4: Chat Endpoint

1. **MISS-01: Scenario comparison not passed to AI context**
   - Handler does not store `ScenarioComparison` result
   - `buildChatSystemPrompt` never populates `input.Scenario`
   - Fix: Add `lastScenario` field to Handler, store in CompareScenario, read in buildChatSystemPrompt
   - Affected: CTX-03 E2E flow

2. **GAP-01: Default AUTH_MODE=optional allows unauthenticated chat access**
   - Auth middleware is applied (CHAT-02 literal requirement met)
   - But `AUTH_MODE=optional` default permits anonymous requests through
   - Rate limiting falls back to per-IP instead of per-user (CHAT-03 semantic gap)
   - Fix: Either require `AUTH_MODE=required` in docs/config or add nil-user check in handler

3. **GAP-02: Empty ChatConfig defaults in main.go**
   - `ai.NewAnthropicProvider(cfg.AIAPIKey, ai.ChatConfig{})` passes zero-value MaxTokens and empty Model
   - Production behavior depends on Anthropic SDK/API accepting these as "use defaults"
   - Fix: Pass explicit defaults like `ai.ChatConfig{MaxTokens: 4096, Model: "claude-sonnet-4-5-20250514"}`
